Programação Orientada a Objetos

1.Conceitos Fundamentais

Classe e Objeto

Classe: É o "molde" ou projeto que define as características e comportamentos de algo (ex: a planta de uma casa).

Objeto: É a instância real da classe, criada a partir do molde (ex: a casa construída).

Atributos: São as características ou dados do objeto (ex: cor, modelo).

Métodos: São as ações ou funções que o objeto pode realizar (ex: ligar, acelerar).

Costrutor: é uma função especial no momento da criação (instaciação) de um objeto. Sua principal função é inicializar os atributos do objeto, garantindo que ele comece com um estado válido e definido.

Herança: Uma determinada classe herda atributos da classe pai.

Abstração: é o processo de simplificar sistemas complexos modelando apenas as características e comportamentos essenciais de um objeto.

Polimorfismo: Permite que objetos de diferentes classes sejam tratados de forma unificada, mas com comportamentos específicos.

Encapsulamento: Ele priva os atributos para melhorar a segurança, evitando acessos indevidos.

Reflexão Técnica

Evitar acúmulo de código que causaria dificuldade caso houver algum erro dentro do código.

Os dados ficam públicos fazendo com que qualquer parte do sistema altere o estado de um objeto, causando comportamentos inesperados.

Herança vira um problema quando o sistema fica muito simplificado.

Linguagem de Programação

1. Estruturas Fundamentais

Variáveis e Tipos de Dados: Variáveis são espaços de memória que guardam valores. Elas podem ser do tipo primitivo ou de referência.

Tipos Primitivos: São os tipos mais básicos da programação:
• int – números inteiros
• float / double – números com casas decimais
• boolean – verdadeiro (true) ou falso (false)
• char – um único caractere

Tipos de Referência: Guardam o endereço de memória de um objeto, como classes, arrays e objetos criados pelo programador.

Estruturas Condicionais: Permitem que o programa tome decisões usando if, else e switch.

Estruturas de Repetição: Permitem repetir blocos de código usando for, while e do-while.

Funções / Métodos: Blocos de código reutilizáveis que executam tarefas específicas.

Tratamento de Erros: Utilizado para evitar que o programa pare ao encontrar falhas, usando mecanismos como try e catch.

Manipulação de Listas/Arrays: Estruturas que armazenam vários valores em uma única variável.

2. Boas Práticas

Padronização de nomes: Utilizar nomes claros e organizados para facilitar a leitura do código.

Organização de arquivos: Separar arquivos por responsabilidade.

Separação de responsabilidades: Cada parte do sistema deve ter uma função específica.

Versionamento com Git: Utilizar controle de versão para registrar alterações no código.

3. Mini Projeto Console

Menu interativo: Interface no console com opções numeradas.

Cadastro simples: Permitir registro de informações básicas.

Validação de dados: Garantir que os dados estejam corretos antes de salvar.

Armazenamento temporário em memória: Utilizar listas ou arrays para guardar os dados enquanto o programa estiver rodando.

Banco de Dados

1. Conceitos

Dado vs Informação: Dado é um valor bruto sem contexto. Informação é o dado processado e organizado com significado.

Banco de Dados e SGBD: Banco de Dados é um conjunto organizado de dados. SGBD é o sistema que gerencia esse banco, como MySQL ou PostgreSQL.

Tabelas, Registros e Campos: Tabela é onde os dados são armazenados. Registro é uma linha da tabela. Campo é uma coluna da tabela.

Chave Primária e Estrangeira: Chave primária identifica um registro de forma única. Chave estrangeira faz a ligação entre tabelas.

Relacionamentos (1:1, 1:N, N:N): Definem como as tabelas se conectam entre si.

Normalização: Processo de organizar as tabelas para evitar redundância e melhorar a integridade dos dados.

2. Modelagem

Modelo Conceitual (DER): Representação gráfica das entidades e seus relacionamentos.

Modelo Lógico: Estrutura detalhada das tabelas com seus campos e tipos.

Script SQL de criação das tabelas: Comandos SQL usados para criar as tabelas no banco de dados.

3. Consultas SQL

SELECT: Consulta dados da tabela.

INSERT: Insere novos registros.

UPDATE: Atualiza registros existentes.

DELETE: Remove registros.

JOIN: Junta dados de duas ou mais tabelas.

WHERE: Filtra registros com base em uma condição.

GROUP BY: Agrupa registros para realizar cálculos.
